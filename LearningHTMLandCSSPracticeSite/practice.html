Ems vs. Percentages vs. Pixels

This is a p with the class "evenlesswide". Ems and percentages are more responsive and preferable in most cases, but pixels are good for images, borders, and positioning.

These are p's that are just the width of the div. I still need to get some clarification on the different types of code used to center things, since the syntax is different depending on the type of element (divs, paragraphs, images, text, etc).
If it's a paragraph, a heading, or text inside a navbar div, use text-align: center;
If it's a div, image, or form: use margin: 0 auto 0 auto;
Or both. For the "evenlesswide" paragraph above, I did text-align: center; to center the text in the paragraph. Then margin: 0 auto 0 auto; to center the paragraph inside the div.

However, there seems to be some inherited styling somewhere that makes the browser think that the width of the horizontal navbar is narrower than it is. But I haven't been able to find it yet, even after comparing my code with the author's.... Hmmm...

Relative and Static Positioning

Both can create some overlap. Relative positioning places elements a certain distance from their normal position. Absolute and fixed/static positioning places elements a certain distance from the edges of the browser window. The div above is getting nudged more to the right with relative positioning.

Background Images

Sample code:
div {
background: url(‘img/flower.jpg’) repeat 10px 10px #ffffe0;
}
Can also use actual image url. Or no-repeat. 10px 10px positions it 10px from both x and y axis. #ffffe0 is the background color for any area the image doesn't cover.
